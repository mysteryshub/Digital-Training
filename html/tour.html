All Features on DigitalHQTraining













User Roles

The User Roles functionality is a fundamental aspect of the Digital HQ application and allows administrators to define which users get which privileges on the application.

Client Owner

There can be only one Client Owner in an organization. The client owner can add and edit categories and change the settings for their organization. Client Owners also have all the privileges of Department Heads, Client Instructors and End Users.

Department Head

There can be multiple Department Heads in an organization. Department Heads can manage users within their own departments and view their reports. They can also edit Category Settings for Categories that they are assigned to.

Client Instructors

There can be multiple Client Instructors in an organization. The Client Instructor creates courses for your organization and tests them to make sure they work correctly. They can also write blog posts.

End Users

These are the users that are actually being trained. End Users can manage their own profiles, access courses, and read blog posts. They can also provide feedback and get access to support for this application.

Profile

Users can access their profile through the sidebar. The profile allows the user to customize and display their personal details, which is useful when identifying and interacting with multiple users. Personal information like their date of birth, designation and bio are displayed on the Profile, and the user can also change their password through their profile page.

User Management

Client Owners and Department Heads can access the User Management section on the sidebar. This section allows administrators to add single users, or import or export bulk user data. User information and settings can be modified, users can be suspended, goals can be assigned and user reports can be accessed from this section.

Course Management

This will most likely be the most frequently used component of the Digital HQ Online Training Platform. Administrators can add new courses and modify settings for existing courses. End Users can access courses that administrators have allowed them to access.

Course Settings

Digital HQ offers administrators with a wide range of settings to ensure access to course content is tightly controlled. Courses can be restricted based on date, number of days or maximum enrollments. Access to a course can also be restricted based on the country that the user is accessing the course from. Administrators can also choose whether the course modules should be accessed sequentially or if users can access any module whenever they like. Courses can be allocated to specific user categories like department, designation or batch, ensuring that only the right user can access the course. Once the course has been published, administrators can decide whom to assign the course as a goal for. Only those users will be able to enroll into the course, while other users will only be able to view the course description and add the course to their wish list.

Course Modules

Each course is made up of chapters and each chapter contains modules. A chapter is simply a means to divide a course into meaningful sections. A module can be any form of media: text, slideshow, video, SCORM, event, discussion or quiz. Each module can be individually edited or relocated within the course. Administrators can attach downloadable files to each module. Users can rate each module based on how effective they felt the module was and administrators can access these scores in the Reporting page.

Discussions

Each course has its own discussion section where users can ask questions and exchange ideas. Course instructors are also encouraged to access the discussions as often as possible to chime in and address concerns. Users can vote on questions they feel are valid, and questions with more votes are displayed more prominently. Users are notified each time there is a new contribution to their discussion.

Bookmarks

The bookmarks feature is accessible by clicking the star on the header and allows users to quickly access important modules by saving them as a bookmark.

Marketplace (Coming Soon)

We are currently in the process of adding thousands of hours of internationally certified training content in subjects like Sales and Marketing, Customer Service, Ethics and Leadership. Courses will be available through our marketplace where clients can simply click to launch any course throughout their organization.

Category Management

Courses, users and blogs can be classified into categories in order to simplify access and tighten control over who sees what. Types of categories include course, level, department, designation, location, division, batch and other. Courses, users and blogs can be assigned to multiple categories.

Reporting

The reporting section can be accessed from the sidebar. The main reporting panel offers insights to administrators about their organization as a whole, including information about how many users are logging in each month, how many new courses are being added each month and how many users are completing their courses each month. The reporting feature in the User Management and Course Management pages gives administrator more specific information like the progress of each user in each course or their scores in tests and exams in each course.

Settings Management

The settings panel can be accessed through the sidebar and allows the user to customize the Digital HQ Online Training Platform to suit their needs and requirements.

Site Settings

Basic site and client information.

Visual Settings

Administrators can customize the look of the application based on their branding requirements.

Notification Settings

SMTP settings along with email and notification settings.

Billing Settings

Limits on new user creation, new course creation and date-based restriction for the whole organization.

Security Settings

Password policy customization, enable forgot password, static IP address and more can be set.

Blog

The Blog, also known as the announcements section, is accessible through the sidebar. Administrators can edit and publish announcements, and assign categories to each post to dictate who is able to see the post.

Notifications

Notifications are system-generated and display in the header under the bell icon. A notification is generally an indication of an external action, such as an administrator assigning a user to a course or another user responding to a comment in a discussion. Administrators can decide which notifications are also emailed to the user by going to Notification Settings.

Alerts

Alerts are system-generated confirmation messages that display in the top right corner of the window, generally in response to a user’s own actions, for example when they change their password or delete a course.

Support

The support section can be found in the sidebar and is a way for the users of a client to communicate with the team and Digital HQ and vice versa. Through this page, users can access FAQ, submit bugs and learn more about the status of the Digital HQ application.
